# Botcoin — AI Agent for $BOTCOIN | https://botcoin.live/

> **An autonomous Twitter/X reply bot powered by xAI (Grok) — the on-chain AI admin of $BOTCOIN on Solana.**

Botcoin monitors mentions of [@getbotcoin](https://x.com/getbotcoin) and replies in real time, in character, using Grok to generate responses. It understands Solana, pump.fun, degen culture, and knows exactly what $BOTCOIN is. Every reply is generated by an AI that *is* the coin.

## How the Bot Works

```
Twitter mention → Poll API (every ~30s)
                      ↓
              Fetch parent tweet (if reply)
                      ↓
            Build context for Grok prompt
                      ↓
          xAI API (3-attempt strategy)
          ├─ Attempt 1: Primary character prompt
          ├─ Attempt 2: Fallback creative-writing frame
          └─ Attempt 3: Hardcoded in-character reply
                      ↓
             Post reply via Twitter API v2
```

The 3-attempt strategy ensures a reply is always posted, even if xAI's filters trigger on the first pass. Each attempt uses a different prompt framing to stay in character while working within content policy.

## Setup

### 1. Clone

```bash
git clone https://github.com/YOUR_USERNAME/botcoin.git
cd botcoin
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and fill in your keys (see [Configuration](#configuration) below).

### 4. Run

```bash
python botcoin.py
```

The bot starts polling immediately and logs every mention, AI response, and reply to the terminal.

## Configuration

All credentials and settings are stored in `.env`. **Never commit this file.**

| Variable | Description |
|---|---|
| `TWITTER_CONSUMER_KEY` | Twitter app consumer key |
| `TWITTER_CONSUMER_SECRET` | Twitter app consumer secret |
| `TWITTER_ACCESS_TOKEN` | OAuth1 access token (your account) |
| `TWITTER_ACCESS_TOKEN_SECRET` | OAuth1 access token secret |
| `XAI_API_KEY` | xAI (Grok) API key from [x.ai](https://x.ai) |
| `TWITTER_USERNAME` | Your Twitter username without `@` |
| `BOTCOIN_CA` | Your pump.fun contract address |

### Twitter API requirements

- **App permissions:** Read + Write (required to post replies)
- **API tier:** Basic or higher (for recent search endpoint)
- **OAuth:** Both OAuth1 (for posting) and OAuth2 bearer (for reading) are used

Get your keys at [developer.x.com](https://developer.x.com).

## Key Design Decisions

**Single-file architecture** — everything lives in `botcoin.py`. No frameworks, no databases, no queues. It's a polling bot: simple to run, simple to audit, simple to fork.

**3-attempt response strategy** — AI models sometimes refuse to engage with crypto topics. The bot tries three progressively softer prompt framings before falling back to hardcoded in-character replies. A reply always goes out.

**Parent tweet context** — when someone tags the bot in a *reply* to another tweet, the bot fetches the original post and includes it in the Grok prompt. This gives responses that are actually contextually relevant, not just generic.

**Stateless** — no database. The `last_checked` timestamp is kept in memory. If the bot restarts, it picks up from 15 minutes ago to avoid missing mentions.
